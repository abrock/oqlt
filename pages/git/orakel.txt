SERVER-SETUP

1. GRUNDEINSTELLUNGEN

$ cat /etc/conf.d/git-daemon
GITDAEMON_OPTS="--syslog --base-path=/mnt/unit0/git /mnt/unit0/git"
GIT_USER="git"
GIT_GROUP="git"

$ mkdir /mnt/unit0/git
$ chown git:git /mnt/unit0/git
$ useradd -d /mnt/unit0/git git
$ /etc/init.d/git-daemon start
$ rc-update add git-daemon default

2. REPO EINRICHTEN

$ su - git

	a) PUBLIC

	$ cd /mnt/unit0/git
	$ GIT_DIR="test.git" git --bare init --shared=group
	$ touch test.git/git-daemon-export-ok

	b) PRIVATE

	$ cd /mnt/unit0/git
	$ GIT_DIR="test.git" git --bare init --shared=group

3. REPO INITIAL BEFÜLLEN (VOM CLIENT AUS)

$ mkdir test
$ cd test
$ git init
$ touch README
$ git add .
$ git commit -m "README angelegt"
$ git push user@kral.dyndns.ws:/mnt/unit0/git/test.git --all

CLIENT-SETUP

1. REPO KLONEN

	a) PUBLIC

	$ git clone git://kral.dyndns.ws/test.git

	b) PRIVATE

	$ git clone user@kral.dyndns.ws:/mnt/unit0/git/test.git

$ cd test

2. GRUNDEINSTELLUNGEN

$ git config user.name "user"
$ git config user.email "user@example.com"

3. WORKFLOW

3.1. Auffrischen

	a) PUBLIC

	$ git pull

	b) PRIVATE

	$ git pull user@kral.dyndns.ws:/mnt/unit0/git/test.git

3.2. Manipulieren

	a) Datei editieren

	b) Datei hinzufügen

	$ git add

	c) Datei/Verzeichnis verschieben/umbenennen

	$ git mv

	d) Datei/Verzeichnis entfernen

	$ git rm

3.3. Kommentieren

$ git commit -a -m "Kommentar"

3.4. Aktualisieren

$ git push user@kral.dyndns.ws:/mnt/unit0/git/test.git
